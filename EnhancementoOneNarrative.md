The artifact I selected for enhancement is my Austin Animal Shelter Dashboard application which was originally developed in CS-340. This project includes a MongoDB-based CRUD system and a dashboard interface used to display and interact with animal shelter data. The application allows users to retrieve, filter, and visualize records from the database. 
I selected this artifact because it demonstrates my ability to work with full-stack components, including database connectivity, data processing, and user interface logic. Additionally, the project provides a strong foundation for enhancement, particularly in the areas of software design, maintainability, and scalability.
For this milestone, I focused on improving the software design and engineering aspects of the application. The original version of the project was developed in a Jupyter-based environment, which was effective for learning purposes but not ideal for long-term maintainability or application development. 
The first major enhancement was transitioning the project into a Visual Studio environment with a structure project layout. I reorganized the codebase into separate directories for application logic, configuration, and supporting components. This improved the overall organization of the project and made it easier to manage and extend. 
I also refactored the database connection logic to remove hardcoded credentials and replace them with a configuration-based approach. Previously, database connection details were embedded directly within the application code, which created security risks and reduced portability. By introducing a dedicated configuration file, I separated concerns and made the application more adaptable to different environments.
Another key enhancement was redesigning the constructor of the AnimalShelter class. In the original implementation, the constructor accepted parameters but immediately overwrote them with hardcoded values, which made the interface misleading and limited its usability. I updated the constructor to properly accept configuration inputs, allowing the database connection to be established dynamically based on external configuration.
In addition, I implemented exception handling within the database connection process. The updated code now attempts to validate the connection using a MongoDB ping command and provides clear error messages if the connection fails. This improvement supports defensive programming practices and ensures that the application fails gracefully instead of silently hanging or crashing.
Finally, I updated the main application entry point to align with the new configuration-driven structure. This included adding runtime feedback messages to confirm successful connections and provide visibility into application behavior during execution.
Through this enhancement process, I gained a deeper understanding of software design principles, particularly the importance of separation of concerns and modularity. Refactoring the project from a Jupyter-based workflow into a structured development environment highlighted how critical organization and clarity are for maintaining and scaling an application. 
I also developed a better application for defensive programming practices. Implementing exception handling in the database connection layer demonstrated how important it is to anticipate failure points and provide meaningful feedback to users or developers. This not only improves reliability but also simplifies debugging and future development. 
One challenge I encountered during this process was transitioning from a course-specific environment to a standalone application environment. The original project relied on a hosted database, which is no longer accessible. As a result, I needed to begin preparing the application for a local MongoDB setup. While the connection logic has been successfully refactored, the next step will be to install and configure a local database instance and populate it with test data. 
Overall, these enhancements represent a significant improvement in the software design and engineering quality of the application. The project is now more modular, maintainable, and aligned with real-world development practices, providing a stronger foundation for continued enhancement throughout the remainder of the capstone.
